방탈출 예약 모니터 Windows 사용방법

1. EscapeRoomMonitor.exe를 더블클릭합니다.
2. 프로그램 창이 뜨고 잠시 뒤 대시보드가 브라우저에서 열립니다.
3. 대시보드의 "전체 매장 오늘 수집"을 누르면 자동 수집 가능한 전국 지점을 확인합니다.
4. "전체 매장 7일 수집"을 누르면 월~일 예약표와 매장별 월 예상매출을 갱신합니다.
5. 7일 수집은 같은 사이트 지점들이 브라우저를 재사용하고 서로 다른 사이트를 최대 8개씩 병렬 확인합니다. 같은 사이트의 다음 매장은 최소 5초 간격을 둡니다.
6. 프로그램 창과 대시보드에서 진행률, 완료 매장 수, 완료 날짜 작업 수와 현재 처리 매장을 확인할 수 있습니다.
7. 화면의 기본 조회일은 한국시간 기준 오늘입니다.
8. 오늘 예약표는 2시간마다, 7일 예약표는 매일 오전 3시 30분에 보충 수집합니다.
9. 프로그램 창을 닫으면 대시보드와 자동 수집도 함께 종료됩니다.
10. 어제 이전 데이터는 기본 조회와 현재 월 예상매출에 포함하지 않습니다.
11. "업계 매장당 평균 월매출"과 "장르별 평균 예상매출"에서 비교 값을 확인합니다.
12. 시작 전에 예약됨을 직접 본 타임은 시간이 지나도 "지나간 확정 예약 기록"에 남습니다.
13. 시간이 지난 뒤 처음 본 비활성 타임은 예약으로 추정하지 않고 확인 불가로 둡니다.
14. 키이스케이프 수동 관측 지점은 업계 평균에 포함되며 "키이스케이프 자료" 탭에서 출처를 확인합니다.

- 네이버 예약에서 숨긴 마감 타임은 예약 완료로 추측하지 않음
- 부분 공개/접근 제한 매장은 예약률 대신 사유 표시
- 셜록홈즈는 설정·수집·통계·기존 기록에서 전부 제외
- 2026-06-15 이후 공개 예약표 실수집 데이터 포함

가격이 0원인 테마는 무료가 아니라 공식 가격 미확인입니다.
예상 매출은 가격이 확인된 예약 완료 슬롯만 계산합니다.
기본 평균 이용 인원은 2.7명이지만 최소 3인 테마는 3인 금액을 적용합니다.
회차 고정가 테마는 인원수로 다시 곱하지 않으며, 추정 인당 부담도 실제 계산 인원으로 나눕니다.
대시보드의 "가격 미반영 예약"은 예약 완료지만 공식 가격이 없어 합계에서 빠진 타임입니다.
월 추정매출(확인분)은 예약표 미수집 매장과 가격 미확인 타임을 제외한 최소 추정치입니다.
확인된 가격에는 출처 URL과 확인일이 함께 표시됩니다.
월 예상매출은 오늘 이후 매장별 월~일 일매출 평균을 해당 월의 요일 수에 적용합니다.
매장별 표에서 월·화·수·목·금·토·일 일매출과 신뢰도를 확인할 수 있습니다.
모든 매출은 공개 인당가와 평균 2.7명을 사용한 추정치이며 실제 결제 매출이 아닙니다.

예약률은 공개 예약 페이지의 판매 상태 스냅샷입니다.
예약률 분모는 시작 전에 상태가 확인된 예약 완료·예약 가능 타임만 사용합니다.
이미 시간이 지난 뒤 처음 확인한 비활성 타임은 예약 완료로 추측하지 않습니다.
플레이33의 "예약 불가"도 시작 전에 확인된 경우에만 예약 완료로 계산합니다.
실제 결제·노쇼 실적은 아닙니다.

로그인, CAPTCHA 우회, 결제, 실제 예약, 비공개 API 호출 기능은 없습니다.
각 사이트의 robots.txt와 이용약관을 사용자가 직접 확인해야 합니다.
